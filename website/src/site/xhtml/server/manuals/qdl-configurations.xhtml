Overview

To use QDL on the server, you must configure it. Fortunately, the configuration is pretty much a standard QDL configurations, merely embedded in the service configuration.

Additional features

You may specify the skipBadModulesOnLoad flag in the QDL element. This tells OA4MP that when deserializing workspaces, bad modules should be skipped (bad being defined as for some strange reason unloadable, such as a missing Java module). This may cause errors later if the module is used, but may also be fine in many cases.

You may also set server-wide scripts to run. These are detailed in scripting hooks
